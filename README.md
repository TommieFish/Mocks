# MockCodeTimeline

## Overview
**MockCodeTimeline** is a repository created to track and document all changes made to code provided in advance for a mock assessment.  
Used is to clearly show how errors are identified, fixed, and improved, while keeping a structured record of answers and independent coding tasks.

This repository acts as:
- A **timeline of edits**
- A **record of error fixes**
- A **revision and reflection tool**
- A **portfolio of mock assessment work**

---

## Version Control & Timeline
- Git commits are used to:
  - Track changes over time
  - Show when errors were fixed
  - Document improvements clearly
    
- Commit messages will describe:
  - The issue
  - The fix
  - Any relevant reasoning
